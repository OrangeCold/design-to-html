# design-to-html

把任意内容做成设计精致、**忠于真实品牌设计语言**的单文件 HTML。

内置 74 个从真实网站提取的设计系统（DESIGN.md），每个含完整的颜色 / 字体 / 间距 / 圆角 / 组件 token 与 Do's & Don'ts。你不用凭空"设计"——先匹配一个真实风格，再严格按它的规范生成。

## 为什么这样设计

让 AI「生成一个漂亮的 HTML 页面」很容易，但结果往往是泛泛的、没有品牌识别度的通用页面。这个 skill 的价值在于输出**有真实设计语言支撑**：颜色、字距、圆角、氛围都来自一个具体品牌的设计系统，且严守它的禁忌。所以质量命脉是「忠于选中风格」，不是「看起来好看就行」。

## 工作流（4 步）

### 第 1 步：理解内容
读懂用户给的内容，判断：
- **类型**：产品落地页 / 文章长文 / 简历作品集 / 数据看板 / 营销站 / 复古趣味页…
- **调性**：严肃专业 / 温暖友好 / 暗黑酷炫 / 奢华高端 / 极简 / 复古…
- **目标受众与明暗偏好**（用户没明说就推断）

### 第 2 步：匹配候选风格
读取 `references/style-index.md`（74 个风格的紧凑索引，匹配只靠它，不要直接翻 design-md/）。沿四个维度语义匹配出 **1–3 个候选**：

1. **内容类型** —— 这是什么场景的页面
2. **调性氛围** —— 暖编辑 / 暗黑酷炫 / 奢华 / 极简 / 复古…
3. **明暗偏好** —— 用户没说就按内容推断（技术·游戏·创意工具常偏暗；编辑·温暖·零售常偏亮）
4. **行业线索** —— 金融 / 汽车 / AI / 电商 / 媒体 关键词直接指向对应行业品牌

匹配靠语义判断，不靠硬编码规则——索引里每行的「调性关键词 / 明暗 / 最适合这些内容 / 标志特征」就是依据。用户若直接点名（「做成 Stripe 那种」「要 Linear 的感觉」），直接锁定该风格，跳到第 4 步。

### 第 3 步：选定风格
- **唯一命中** → 直接用，告诉用户选了哪个、为什么。
- **多个都说得通** → 用 AskUserQuestion 给用户做单选。每个选项描述带上「调性 + 主色 + 明暗 + 适合理由」，让用户一眼选对，例如：
  > **Linear** — 近黑画布 + 薰衣草蓝，工程师向，冷峻精密

### 第 4 步：生成 HTML
读取 `references/design-md/<slug>.md`（**只读选中的那一个**，不要多读），再读 `references/html-generation-guide.md`，按其规范生成**单文件自包含 HTML**：

- CSS 全内联；字体引入该风格推荐的免费 Google Fonts + 完整 fallback 栈
- 颜色 / 字号 / 字距 / 圆角 / 间距一律用 DESIGN.md 的真实 token，不自创近似值
- **逐条遵守该风格的 Do's & Don'ts**——这是与通用页面的核心区别
- 按内容智能选择页面结构（hero / feature grid / 文章排版 / 数据卡…）
- 按 DESIGN.md 的 Responsive Behavior 做响应式断点

生成后过一遍 `html-generation-guide.md` 的自检清单，再交付。

## 关键原则

- **先匹配，后生成**：永远先确定一个具体风格，再动手写 CSS。不要在没选定风格时凭感觉发挥。
- **忠于 token，不自创**：颜色用文件里的 hex，尺度用 token。近似值会破坏品牌识别度。
- **Do's & Don'ts 是硬约束**：违反 Don't 的元素，再好看也要删。
- **按需加载**：只读匹配到的那个 DESIGN.md，不要一次性读多个文件。
